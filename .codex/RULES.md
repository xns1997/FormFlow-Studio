# FormFlow Studio 项目铁律

1. 不要 hero 标题。
   - 不要为了“更像产品页”额外加入大标题、大片副标题、横幅式说明。
   - 页面已有信息架构足够时，禁止再叠加视觉主标题。

2. 不要在 UI 里乱加没必要的提示文案。
   - 禁止加入类似“可继续拖入 / 拖出”“当前已有 N 个子控件”“这里是内容区域”这类解释型废话。
   - 交互应尽量通过布局、层级、状态、边框、留白、hover、disabled、selection 等视觉语言表达，而不是靠额外文字说明。

3. 提示文案只在以下情况允许出现：
   - 用户当前无法继续操作，且必须知道原因。
   - 操作具有不可逆或高风险后果。
   - 空态会让用户完全失去方向，且无法通过结构本身理解页面用途。

4. 默认偏向“少字、弱提示、强结构”。
   - 能不写就不写。
   - 能做成隐性状态，就不要做成显性标签。
   - 能靠结构表达，就不要靠说明表达。
