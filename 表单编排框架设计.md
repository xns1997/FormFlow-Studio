下面给一个**独立于 Excel 的表单编排框架设计方案**。它不是 Excel 插件，而是一个独立应用/网页系统：Excel 只是数据源之一，加载后被解析成结构化数据，再由用户设计 UI、绑定字段、定义行为、测试并发布。

---

# 一、框架定位

## 框架名称示例

**FormFlow Studio**

一句话定位：

> 一个面向 Excel / CSV / 数据表的可视化表单编排框架，支持用户读取表格数据、设计表单界面、通过连线绑定字段，并通过 JS 编辑器与节点编辑器定义动态行为。

核心目标：

1. **Excel 只作为数据源，不依赖 Excel 运行环境**
2. **表单 UI 可视化编排**
3. **字段绑定通过连线完成**
4. **表单行为可通过节点图或 JS 脚本定义**
5. **支持数据预览、设计、绑定、行为、测试、设置的完整工作流**
6. **最终可导出为独立表单应用、JSON 配置或运行时包**

# 三、核心设计原则

## 1. Excel 文件只是输入，不是运行依赖

加载 Excel 后，系统会把它转换成自己的内部数据结构。

例如：

```
Excel 文件
  ↓
WorkbookModel
  ↓
SheetModel
  ↓
ColumnSchema
  ↓
RowData
```

系统不直接依赖 Excel 的界面、公式、宏或插件能力。

---

## 2. 表单设计与数据结构解耦

用户设计的表单不直接等于 Excel 表格。

表单中可以有：

```
输入框
下拉框
日期选择器
表格子组件
图片上传
文件上传
评分组件
开关
分组容器
多步骤表单
弹窗
动态列表
```

这些 UI 组件通过**连接关系**绑定到 Excel 的某一列或某些列。

---

## 3. 字段绑定不是写死配置，而是图结构

绑定关系可以理解为一张图：

```
[姓名输入框.value] ─────▶ [Sheet1.姓名列]
[年龄输入框.value] ─────▶ [Sheet1.年龄列]
[部门下拉框.value] ────▶ [Sheet1.部门列]
```

每个 UI 组件和每个数据列都有连接点，也就是 Port。

---

## 4. 行为定义分成两种模式

### 简单用户使用节点编辑器

适合非程序用户。

例如：

```
当【部门】变更
  如果 部门 = 技术部
    显示【技术栈】字段
  否则
    隐藏【技术栈】字段
```

### 高级用户使用 JS 编辑器

适合开发者。

例如可以定义：

```
onChange("department", (value, ctx) => {
  ctx.setVisible("techStack", value === "技术部")
})
```

节点编辑器和 JS 编辑器可以互相补充，不要求完全互转。

## 5. 支持三种绑定模式

### 单字段绑定

```

一个组件绑定多列。

例如地址组件：

```

```

```

# 十三、绑定关系的工作方式

假设 Excel 中有一列：

```
Sheet: 员工信息
Column: 部门
```

用户在表单中放了一个下拉框：

```

Port: value
```

连接后形成：

```
departmentSelect.value ⇄ 员工信息.部门
```

运行时：

```
读取当前行的 部门 值
  ↓
写入 departmentSelect.value
  ↓
用户修改下拉框
  ↓
更新 formValues.department
  ↓
同步到数据变更 patch
```

最终提交结果：

```
{
  "sheet": "员工信息",
  "rowIndex": 5,
  "changes": {
    "部门": {
      "oldValue": "销售部",

```

# 十四、节点编辑器与 JS 编辑器的关系

## 推荐设计

节点编辑器负责大部分普通逻辑。

JS 编辑器负责：

```
复杂计算
复杂校验
跨 Sheet 查询
接口请求
批量处理
自定义格式化
复杂联动
```

---

## 二者不是完全互相替代

节点编辑器保存为：

```
BehaviorGraph
```

JS 编辑器保存为：

```
ScriptModule
```

运行时统一编译为：

```
RuntimeAction
```

执行时都进入同一个行为引擎。

```
节点行为 ─┐
          ├─ 行为引擎 ─▶ 修改表单状态 / 数据状态
JS 脚本 ──┘
```

# 二十四、这个框架的核心价值

它本质上不是“Excel 表单美化器”，而是：

> 把 Excel 这种二维数据源，转换成一个可设计、可绑定、可编排、可测试、可发布的独立表单应用。

核心抽象是：

```
数据源 DataSource
字段 Column
组件 Component
连接 Binding
行为 Behavior
运行时 Runtime
```

只要这几个抽象稳定，后面 Excel 可以替换成数据库，表单可以导出成 Web 页面，行为可以接入工作流，整个系统就不会被 Excel 绑死。
